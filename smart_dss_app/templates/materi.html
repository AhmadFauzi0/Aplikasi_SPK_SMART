Materi & Formula Metode SMART

1. Landasan Multi-Attribute Utility Theory (MAUT)

Berdasarkan rumusan dasar teori utilitas multi-atribut, preferensi skor total $V(a_i)$ dari sebuah alternatif dihitung menggunakan agregasi perkalian antara bobot kriteria dan nilai utilitasnya:

$$V(a_i) = \sum_{j=1}^{n} w_j \cdot u_j(x_{ij})$$
Contoh Implementasi: Jika Vendor A memiliki utilitas Biaya 0.435 (bobot 0.171) dan utilitas Fitur 1.0 (bobot 0.257), maka skor agregat sementaranya adalah: $(0.171 \times 0.435) + (0.257 \times 1.0) = 0.074 + 0.257 = 0.331$.
2. Transformasi Utilitas Linear Skala $[0,1]$

Nilai asli performa harus dinormalisasi menjadi rentang 0 (terburuk) hingga 1 (terbaik). Caranya berbeda tergantung sifat kriteria:

Kriteria Benefit (Keuntungan)

Makin besar nilai asli, makin baik.

$$u_{ij} = \frac{x_{ij} - x_{j}^{\min}}{x_{j}^{\max} - x_{j}^{\min}}$$

Contoh (Kemudahan Penggunaan):
Min = 6, Max = 9. Jika skor Vendor A = 8.
$u = \frac{8 - 6}{9 - 6} = \frac{2}{3} = \mathbf{0.667}$

Kriteria Cost (Biaya)

Makin kecil nilai asli, makin baik.

$$u_{ij} = \frac{x_{j}^{\max} - x_{ij}}{x_{j}^{\max} - x_{j}^{\min}}$$

Contoh (Harga/Biaya):
Min = 120, Max = 350. Jika Harga Vendor A = 250.
$u = \frac{350 - 250}{350 - 120} = \frac{100}{230} = \mathbf{0.435}$

3. Metode Elicitasi & Normalisasi Bobot Kriteria ($w_j$)
  • Direct Rating / Swing Weighting
    Membagi input skala nilai dengan total nilai seluruh kriteria $\sum w_j = 1$.
    $$w_j = \frac{R_j}{\sum_{k=1}^{n} R_k}$$
    Contoh: Pengambil keputusan memberi rating {Biaya: 60, Fitur: 90}. Total = 150.
    Bobot Biaya = $\frac{60}{150} = \mathbf{0.40}$, Bobot Fitur = $\frac{90}{150} = \mathbf{0.60}$.
  • Rank Order Centroid (ROC)
    Mengubah peringkat kepentingan ($k=1$ paling penting) menjadi bobot non-linear.
    $$w_k = \frac{1}{n} \sum_{j=k}^{n} \frac{1}{j}$$
    Contoh (Jika ada 3 kriteria, $n=3$):
    Peringkat 1 ($k=1$): $w_1 = \frac{1}{3} \times (\frac{1}{1} + \frac{1}{2} + \frac{1}{3}) = \mathbf{0.611}$
    Peringkat 2 ($k=2$): $w_2 = \frac{1}{3} \times (\frac{1}{2} + \frac{1}{3}) = \mathbf{0.278}$
    Peringkat 3 ($k=3$): $w_3 = \frac{1}{3} \times (\frac{1}{3}) = \mathbf{0.111}$
4. Analisis Sensitivitas OAT (One-At-a-Time)

Digunakan untuk menguji kekuatan (robustness) keputusan. Ketika bobot kriteria utama ($k$) diubah paksa ($w_k^{new}$), bobot kriteria lainnya ($j \neq k$) harus diredistribusikan proporsional menggunakan formula:

$$w_j^{new} = w_j^{base} \cdot \frac{1 - w_k^{new}}{1 - w_k^{base}}$$
Contoh Implementasi:
Awalnya bobot Biaya ($w_k^{base}$) = 0.171 dan bobot Fitur ($w_j^{base}$) = 0.257.
Jika dalam simulasi sensitivitas kita menaikkan bobot Biaya secara radikal menjadi 53% ($w_k^{new} = 0.530$), berapakah bobot Fitur yang baru?
$w_j^{new} = 0.257 \times \frac{1 - 0.530}{1 - 0.171} = 0.257 \times \frac{0.470}{0.829} = 0.257 \times 0.567 = \mathbf{0.145}$
(Porsi bobot fitur otomatis mengecil karena porsi biaya dibesarkan, namun rasio kepentingannya terhadap kriteria lain tetap stabil).
